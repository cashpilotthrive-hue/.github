# Security Policy

## Reporting Security Vulnerabilities

If you discover a security issue in this repo, please submit it through the [GitHub Security Bug Bounty](https://hackerone.com/github)

**Please do not create public issues for security vulnerabilities.**

Thanks for helping make GitHub Actions safe for everyone.

## Security Best Practices

When using this Linux system setup:

1. **Keep Systems Updated**: Regularly apply security patches
2. **Use Strong Authentication**: Enable SSH keys and disable password auth
3. **Minimize Attack Surface**: Only install necessary packages
4. **Monitor for Vulnerabilities**: Use automated scanning tools
5. **Follow Principle of Least Privilege**: Don't run services as root
6. **Enable Firewalls**: Configure UFW or firewalld
7. **Audit System Access**: Review logs regularly

## Supported Versions

Security updates are provided for:
- Latest main branch
- Current release tags
- Active development branches

## Security Contact

For security-related questions or concerns:
- Create a security advisory through GitHub
- Contact repository maintainers privately
- Follow responsible disclosure practices
